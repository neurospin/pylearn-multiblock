   3  The :mod:`parsimony.algorithms` module includes several algorithms used 
   4  throughout the package. 
   5   
   6  Algorithms may not store states. I.e., if they are classes, do not keep 
   7  references to objects with state in the algorithm objects. It should be 
   8  possible to copy and share algorithms between estimators, and thus they should 
   9  not depend on any state. 
  10   
  11  There are currently two types of algorithms: implicit and explicit. The 
  12  difference is whether they run directly on the data (implicit) or if they have 
  13  an actual loss function than is minimised (explicit). Implicit algorithms take 
  14  the data as input, and then run on the data. Explicit algorithms take a loss 
  15  function and a start vector as input, and then minimise the function value 
  16  starting from the point of the start vector.
